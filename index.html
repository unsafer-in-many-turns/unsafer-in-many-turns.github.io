Why Multi-Turn Agent Safety?

Existing safety benchmarks test either single-turn tool agents or multi-turn chat, never both. As agents engage in multi-turn interactions with diverse tools, harmful intent can be distributed across benign-looking turns, evading current safety mechanisms.

Multi-Turn Attack Taxonomy

We propose a principled taxonomy that decomposes single-turn harmful tasks into multi-turn attack sequences across 3 dimensions:

Format
How the transformation is structured (Addition vs. Decomposition)
Method
How it is performed (Mapping, Wrapping, Composition, Identity)
Target
What is manipulated (Data Files vs. Environment States)

This yields 8 attack subcategories. The taxonomy preserves the same harmful objective while distributing it across multiple benign-appearing turns.

ToolShield: Self-Exploration Defense

A training-free, tool-agnostic defense that leverages the agent's own capabilities. When encountering a new tool, the agent:

  1. Generates test cases through structured risk reasoning
  2. Executes them in a sandbox to observe behavior
  3. Distills safety experiences from execution traces
  4. Updates and applies experiences during deployment

No weight updates. No tool-specific engineering. Just leveraging the agent's own capability to learn safe behavior through self-exploration.
